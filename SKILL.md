# PRD 信息提取规范

执行本规范时，先读取 `references/rules.md`，再按本文的输出格式生成 PRD 事实清单。

## 目标
从 PRD、活动需求或玩法说明中提取首页设计所需的事实信息，供下一步 `games-homepage-layout` 判断页面骨架、槽位状态和布局关系。

本阶段只回答“PRD 说了什么”，不回答“页面怎么排”。
提取结果必须交给用户确认，确认后才能进入 `games-homepage-layout`。

## 强约束
1. 不在本阶段选择全屏、半屏或三分屏骨架。
2. 不在本阶段决定槽位是否出现、组合关系、尺寸、间距或位置。
3. 不根据经验补充 PRD 中不存在的模块。
4. 缺失信息填写“待补充”，不得自行编造。

## 验收标准
1. 玩法动作、对象、过程和结果是否可用于绘制交互稿。
2. 所有信息是否来自 PRD，而非设计阶段自行补充。
3. 是否区分“明确要求”“未提及”和“待确认”。
4. 是否为下一步首页布局判断提供足够信息。
5. 是否已获得用户确认；未确认时不得进入下一阶段。
