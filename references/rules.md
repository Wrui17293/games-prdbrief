# PRD 事实提取规则

## 目标

只提取 PRD、活动需求或玩法说明中的事实信息，为后续首页布局判断提供输入。

## 核心规则

1. 只回答“PRD 说了什么”，不回答“页面怎么排”。
2. 不选择全屏、半屏或三分屏骨架。
3. 不决定槽位是否出现、组合关系、尺寸、间距或位置。
4. 不根据经验补充 PRD 中没有写明的模块。
5. 缺失信息填写“待补充”，不得自行编造。
6. 明确区分“明确要求”“未提及”“待确认”“需求冲突”。
7. 输出事实清单后必须等待用户确认；未确认前不得进入 `games-homepage-layout`。

## 必须覆盖的信息

- 页面目标：核心目标、核心转化动作、进入来源、完成后去向。
- 活动信息：活动名称、主题利益点、时间、说明、规则入口。
- 核心玩法：玩法名称、目标、表现载体、用户动作、操作对象、过程、结果和状态。
- 用户状态：账户、等级、体力、次数、资产、常驻展示信息和入口。
- 任务奖励：任务、奖励、累计进度、阶段节点、倒计时、助力分享。
- 操作信息：主操作、辅操作、其他入口、禁用条件。
- 内容承接：资源位、Feeds 和承接目的。
- 页面状态：默认态、进行态、完成态、失败态、无次数或异常态。

## 验收标准

1. 信息均来自 PRD 或用户补充，不来自设计推断。
2. 玩法动作、对象、过程和结果足够支持后续低保真表达。
3. 缺失项和冲突项已经列出。
4. 用户确认状态明确。
