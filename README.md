# games-prdbrief

- 提交人：xiashiyu.09
- 维护人：xiashiyu.09

- 所属业务域：营销域
- 当前状态：进行中

## 用途

这个 Skill 用于从营销互动活动 PRD、活动需求或玩法说明中提取首页设计事实清单，帮助后续低保真布局阶段明确页面目标、玩法信息、任务奖励、操作状态、内容承接、缺失冲突和用户确认结果。

## 适用场景

- 用户提供活动 PRD、玩法说明或营销需求，需要先整理成结构化事实清单。
- 低保真布局前，需要确认 PRD 中哪些信息是明确需求、哪些是未提及、哪些需要用户补充。
- 需要避免设计阶段直接脑补页面模块、槽位、尺寸或布局关系。

## 输入

- 活动 PRD、活动需求、玩法说明或用户口头描述。
- 活动名称、主题利益点、活动时间、规则入口等基础信息。
- 核心玩法、用户动作、操作对象、奖励结果、页面状态等业务信息。

## 输出

- PRD 事实清单。
- 缺失信息清单。
- 需求冲突清单。
- 需要用户确认的问题。
- 用户确认后的最终事实记录。

## 依赖资料

- `SKILL.md`：PRD 事实提取规则与输出格式。
- `references/rules.md`：事实提取边界、确认门禁和验收规则。
- 用户提供的 PRD、活动需求或玩法说明。

## 边界和风险

- 不在本阶段判断页面骨架、槽位状态、组合关系、尺寸、间距或位置。
- 不根据经验补充 PRD 中不存在的模块。
- 缺失信息必须标记为“待补充”，不能自行编造。
- 未经用户确认的事实清单不能进入 `games-homepage-layout`。
- 涉及真实业务数据、活动成本、奖励规则、法务规则时必须由业务方人工审核。
